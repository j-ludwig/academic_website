16.1.1 Univariate Statistics

In this section, we present univariate statistics across different regions for CPI inflation, real home price inflation, and real policy rates. In the context of statistics, the term “univariate” refers to the analysis of a single variable without consideration of any other variables. Metrics like mean, standard deviation, minimum, and maximum are univariate statistics because they describe features of one variable in isolation. They summarize the central tendency, variability, and range of possible values for that single variable.

In contrast, associative statistics examine relationships between two or more variables, which will be discussed in the next section. Correlation coefficients, covariances, and regression coefficients are examples of associative statistics. These metrics help us understand how variables interact with each other or how one variable might predict another.

For the univariate statistics, we group the data by region, variable, and measure, then calculate summary statistics for each group, consisting of the time span, mean, and volatility.

# Compute univariate statistics
BIS_univariate_stats <- BIS_core %>% 
    group_by(Indicator, Region) %>% 
    summarize(
        `Time Span` = paste(range(Period), collapse = " - "),
        Mean = mean(Value, na.rm = TRUE),
        Volatility = sd(Value, na.rm = TRUE)
    ) %>% 
    ungroup()

16.1.2 Associative Statistics

This section presents a framework for studying multivariate relationships in economic data, focusing on associative statistics. We explore three specific types of these statistics using the panel data from the Bank for International Settlements Bank for International Settlements (2023b). A more generalized form is also introduced as a fourth category:

  1. Relationships Between Regions by Indicator Across Time Periods: This type assesses how a single indicator, like inflation, correlates between different regions. The variation of the data over time periods is used to estimate these relationships. For example, this approach can reveal how inflation in the United States correlates with inflation in the Euro area.

  2. Relationships Between Indicators by Region Across Time Periods: Unlike the first type, this analysis narrows down to a single region and explores the relationships between different economic indicators within that particular area. Again, temporal variations are use to estimate these relationships. For example, within the United States, we could explore how inflation correlates with policy rates.

  3. Relationships Between Indicators Across Time Periods & Regions: This approach examines the general relationship between different indicators, such as home price inflation and policy rates, by considering data over time from all regions. What sets this apart from the second type is that it uses not only temporal variations but also regional variations to estimate the relationships. For example, it could assess how home price inflation correlates with policy rates using data from multiple time periods and regions.

  4. Relationships Between \(A\) by \(B\) Across \(C\): Note that there are many more ways to estimate associative statistics, where \(A\), \(B\), and \(C\) can contain any combination of “Indicators”, “Regions”, and “Time Periods.” \(B\) may be omitted, but at least one category must be present in \(A\) and \(C\). For example, to compute the covariance for each pair \(\{i, j\} \in A \times A\) for a given \(k\in B\) across \(l\in C\), the formula would be: \[ \text{Cov}_{ij}^{(k)} = \frac{1}{n-1} \sum_{l=1}^{n} \left( X_{il}^{(k)} - \bar{X}_{i}^{(k)} \right) \left( X_{jl}^{(k)} - \bar{X}_{j}^{(k)} \right), \qquad \bar{X}_{i}^{(k)} = \frac{1}{n} \sum_{l=1}^{n} X_{il}^{(k)} \] Here, \(n\) is the number of observations, i.e., the number of elements in \(C\). \(\bar{X}_i^{(k)}\) and \(\bar{X}_j^{(k)}\) are the means of variables at \(i \in A\) and \(j \in A\) for a given \(k\in B\). Finally, \(X_{il}^{(k)}\) and \(X_{jl}^{(k)}\) are the individual observations at \(l\in C\). For example, when computing covariances between regions (\(A\)) by indicators (\(B\)) across time periods (\(C\)), then \(\{i, j\} \in A \times A\) represents a pair of regions - such as the United States and the Euro area - where \(A \times A\) is the set of all possible pairs of regions. Meanwhile, \(k \in B\) specifies an indicator like inflation, with \(B\) being the set of all indicators. Lastly, \(l \in C\) denotes a time period, and \(C\) is the set of all time periods.

Relationships Between Regions by Indicator Across Time Periods

The data typically needs to be reshaped first before the relationships between \(A\) by \(B\) across \(C\) can be computed and visualized. Each element in \(C\) represents an observation, e.g. a time period, whereas the elements in \(A\) and \(B\) represent different variables, e.g. inflation in the United States, or policy rates in the Euro area.

In a two-dimensional array such as a matrix or a data frame, one column represents each element in \(C\) as a separate row, and the data of each pair in \(A\) needs to be organized next to each other as separate columns so that their relationship can be computed and visualized. For example, here, since \(C\) represents time periods, there is a column representing time periods, and since \(A\) represents regions, at least two columns need to be filled with data representing each possible pair of regions.

When the data is in wide format, then there are as many rows as elements in \(C\), and the columns represent all possible combinations of \(A\) and \(B\). Here, each row is a different period, saved in a column called Period, and the columns represent all region and indicator combinations, saved in columns called CPI Inflation.United States, CPI Inflation.Euro area, RRP Inflation.United States, etc. In that case, the relationships can be computed by accessing the columns directly, e.g. cor(CPI Inflation.United States, CPI Inflation.Euro area) computes the correlation of CPI inflation between the two regions.

When the data is in long format, then one column represents the names in \(C\), two additional columns represent the names in \(A\) and \(B\), and the last column captures the values of the data for each combination of \(A\), \(B\), and \(C\). Hence, the data needs to be reshaped first so that there are two columns with data values instead of just one, representing all pairs in \(A\times A\). To do this, the names of \(B\) are spread out across two instead of one column, representing each possible combination. Then there are two data columns representing the data of each of the pair. Hence, in this example, a Period column represents the time periods in \(C\), an Indicator column represents the names in \(B\), the two columns Region X and Region Y represent all name combinations of \(B\), and the data is stored in columns Value X and Value Y. Note that this will make the object even longer. For example, if \(B\) has four elements, then there are \(\binom{4}{2} = 6\) possible combinations of pairs, computed in R as choose(4, 2).

paste_and <- function(x, collapse = ", ", collapse_last = ", and ", 
                      collapse_2_or_less = gsub(",", "", collapse_last)) {
    if(length(x) <= 2) return(paste(x, collapse = collapse_2_or_less))
    paste0(paste(x[-length(x)], collapse = collapse), collapse_last, x[length(x)])
}
